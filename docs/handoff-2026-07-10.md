## What is implemented

- Default independent artifact contract: Plaud supplies metadata and raw audio only;
  imported Plaud transcripts/notes are preserved and labelled but cannot satisfy the
  primary pipeline.
- Idempotent legacy preparation preserves imports, relabels unsafe legacy summaries,
  removes non-provenanced indexes, and requeues usable audio for local ASR.
- Durable stage runs for convert/transcribe/diarize/summarize/index include attempts,
  status, provider/model, source, timestamps, detail, and errors. Core ASR failures
  remain `error`; optional failures retain artifacts as `partial`. Web UI exposes
  Resume versus Rebuild all plus stage diagnostics.
- Ollama `bge-m3` embeddings use `/api/embed` with legacy fallback, model-aware health,
  and stored model provenance. A live two-vector/1024-dimension smoke test passed.
- Apple Silicon ASR is `mlx-community/whisper-large-v3-turbo`. NumPy is constrained
  below 2.5 for numba compatibility. The 1.61 GB model is cached; a local Metal smoke
  test produced word timestamps successfully.
- Pyannote integration targets current open-source
  `pyannote/speaker-diarization-community-1` and supports its v4 output. Package
  `pyannote.audio==4.0.7` is installed locally. Real model use is blocked only by the
  gated-model acceptance/token described below.
- Daemon processing is newest-first and bounded by
  `pipeline.files_per_cycle = 1`; explicit `localplaud work --once` drains the full
  backlog.
- Long transcripts are no longer truncated at 24,000 characters. Summarization maps
  every bounded chunk to coverage notes, recursively consolidates when necessary,
  then applies the selected final template. Coverage metadata is stored in the
  summarize stage detail.
